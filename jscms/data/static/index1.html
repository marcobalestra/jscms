La casa del cuoco

This is a fresh new install of JSCMS.

Content edit is web-based, performed using the browser, while the configuration is done by editing the files shipped with the distribution.

You can have pages of as many types you decide, and they can be indexed (have an ID) or single (like the homepage, pagetype: "index"). The default homepage type, "index", is mandatory.

Root-level HTML content

    • Feel free to customize the index.html page located at site document root level.
      Customize with as many elements you need, elements that can also be referred in template info to decide their content.
    • Custom root-level pages (including HTML node and all the necessary scripts and CSS requests) can be defined at document root level. “index.html” is the default, unless “pagetype.html” is available.
    • Each toplevel homepage, that you can customize and make different depending on the pagetype, is expected to contain a node with ID “jcToplevelContainer” where data are rendered.

Pages content (templates)

Each page type content is defined by a JSON descriptor, defined in the javascript file /jscms/templates/info/pagetype.js

In the template the “html” property specifies the HTML file that will be appended to jcToplevelContainer node. This file is located in the “html” directory /jscms/templates/html/htmlprop.html, and defaults to “index”.

The “content” property specifies the content of the different regions of the page:

    • template.content is an Array of Objects.
    • Each template.content.entry object should have a ”selector“ attribute, a CSS selector indicating where the entry content will be rendered.
      template.content.entry.selector defaults to “#jcToplevelContainer”.

Given the selector property, each template.content.entry can be:

    • A reference to an external content:
      entry.type : "part" and entry.content: partname
      Parts are located in /jscms/templates/parts/ folder.
      entry.part: partname is a valid shortcut (with only one property).
    • Hidden, hiding the area:
      entry.hidden : true — or entry.content: false
    • The rendered content of actual page data, using blocks.
      Each data entry can be a single value (block) or a mixed content of blocks (mixed).
      entry.type : "blocks"
      entry.blocks : Array of Objects.
      Each entry.blocks.entry will be placed in sequence in the area of the above selector, and will specify:
      • a type (eg: “text”, “html”, “mixed)
      • a prop (the property of data file object holding the related values)
      • further attributes depending on type (e.g.: wrap for type:"text")
