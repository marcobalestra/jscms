This is the home page of a new opensource Javascript-based CMS, currently under development.

It will require only a minimal server-side effort, while data storage will rely on JSON files.

Key factors

JSCMS can run WITHOUT any type of serverside language, using simply EcmaScript6 and Apache 2.4 DAV features.

  • Doesn’t require MySQL, Oracle, MariaDB, Postgres nor any other DB
  • Doesn’t require PHP
  • Doesn’t require Node.js
  • Doesn’t require Java
  • Doesn’t require Perl, Python, Ruby or any other scripting language
  • Doens’t require any language or library dependency

Yes, this means that the CMS on your server is made by static files.

Please note: every brand here cited (to clearly state that it's not used) is property of their respective owner(s) - if any.

Overview

The CMS in entirely based on client-side javascript.

It uses the WebDAV features of Apache2 to perform operations, and saves data in JSON static files. Images and attachments are linked to the individual page, and are uploaded as binary files always using WebDAV features.

The CMS is a SPA, a single-page application.
